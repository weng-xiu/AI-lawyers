## 系统重启计划

### 1. 停止当前运行的服务
- **停止后端服务**：使用 `StopCommand` 工具停止当前运行的后端服务
- **停止管理前端**：使用 `StopCommand` 工具停止当前运行的管理前端服务
- **停止用户前端**：使用 `StopCommand` 工具停止当前运行的用户前端服务

### 2. 重新启动后端服务
- **模块**：ai-admin
- **启动方式**：使用 Maven 运行 spring-boot:run 命令
- **命令**：`mvn spring-boot:run`
- **工作目录**：`d:\project_job\AI-lawyers\ai-admin`
- **预期结果**：后端服务启动成功，控制台显示启动成功信息
- **验证方式**：查看控制台日志，确认服务启动成功

### 3. 启动管理前端
- **模块**：ai-ui
- **启动方式**：使用 npm 运行 dev 命令
- **命令**：`npm run dev`
- **工作目录**：`d:\project_job\AI-lawyers\ai-ui`
- **预期结果**：管理前端启动成功，控制台显示访问地址（如 http://localhost:80）
- **验证方式**：使用 curl 命令访问管理前端地址，确认返回 200 状态码

### 4. 启动用户前端
- **模块**：aiuser-ui
- **启动方式**：使用 npm 运行 dev 命令
- **命令**：`npm run dev`
- **工作目录**：`d:\project_job\AI-lawyers\aiuser-ui`
- **预期结果**：用户前端启动成功，控制台显示访问地址（如 http://localhost:81）
- **验证方式**：使用 curl 命令访问用户前端地址，确认返回 200 状态码

### 5. 监控与验证
- 监控各服务的启动日志，确保无错误信息输出
- 验证所有服务均能正常响应请求
- 检查管理端和用户端接口是否正确分离

### 6. 注意事项
- 严格按照顺序启动服务，确保后端服务完全启动后再启动前端服务
- 等待管理前端完全加载并可正常访问后，再启动用户前端应用
- 启动过程中密切关注控制台输出，及时处理可能出现的错误

### 7. 回滚方案
- 如果后端服务启动失败，检查配置文件和依赖，修复问题后重新启动
- 如果前端服务启动失败，检查依赖和配置，修复问题后重新启动
- 如果出现端口冲突，修改配置文件中的端口号后重新启动
